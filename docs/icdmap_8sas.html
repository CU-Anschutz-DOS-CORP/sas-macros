Maps ICD-9 to ICD-10 and vice-versa. More...

Detailed Description

Purpose
This macro uses CMS GEMs and ICD code lists to map ICD-9 to ICD-10 and ICD-10 to ICD-9 diagnosis or procedure codes. It can also be used to generate a list of select codes with descriptions and can also provide a list of code changes for ICD-10 codes.
Author
Rocio Lopez
Date
12-10-2020
Version
SAS 9.2 or later
Usage
A call to this macro has some or all of these parameters (defaults given):
%icdmap(
libname =
,icd9 =
,icd10 =
,codetype = dx
,map = 1
,checkconversion10 = 0
);
Returns
One or more tables printed to the results window.
Parameters
libNameLibrary name assigned to the directory where the CMS tables are stored
icd9List of ICD-9 codes to map, each enclosed by ''
This expression will be used with the IN () operator
icd10List of ICD-10 codes to map, each enclosed by ''
This expression will be used with the IN () operator
codeTypeType of codes provided
Valid options are: DX for diagnosis (default) or PRC for procedures
mapLogical for whether or not to map across versions
Valid options are: 0 for No, 1 for Yes (default)
checkConversion10If ICD-10 list given, print a list of changes in specified codes, if any
Valid options are: 0 for No, 1 for Yes (default)
Note
  • User must have necessary CMS tables stored as .sas7bdat files and connect to the location using LIBNAME statement. Go to https://github.com/lopezr/public-files/blob/main/icdMap/CMS%20ICD%20Tables%20Used%20by%20%25icdMap%20SAS%20macro.pdf to see a list of the tables we use. You may need to update the table and/or column names within the macro depending on your set-up.
  • You can omit the periods from the codes.
  • If you have a full range of codes to use (456.xx, for example), you can query the tables prior to using the macro and create a macro list containing the codes. See Example 3 below.
Example(s)
*Example 1 - Mapping ICD-9-CM to ICD-10-CM ;
%icdmap(
libname = cms
,icd9 = "456.0" "456.20"
);
*Example 2 - Mapping ICD-9-CM to ICD-10-CM and vice vers;
%icdmap(
libname = cms
,icd9 = "456.0" "456.20"
,icd10 = "I85.01" "I85.11"
) ;
*Example 3 - Creating a macro variable with list of codes corresponding to a range ;
*Create list of all 456.xx ICD-9-CM codes;
proc sql noprint;
select compress(ICD9_CODE, " .")
into :list456 separated by '" "'
from cms.CMS_ICD9_DX_DESC_2014
where substr(ICD9_CODE, 1, 3) eq "456"
order by ICD9_CODE;
quit ;
%put ***LIST: "&list456"***;
*Map ICD-9-CM 456.xx to ICD-10-CM" ;
%icdmap(
libname = cms
,icd9 = "&list456") ;
