Detailed Description

Purpose
Runs PROC REPORT and creates a simple report-ready table
Author
Rocio Lopez
Date
02-01-2016
Version
SAS 9.2 or later
Usage
A call to this macro has some or all of these parameters (defaults given):
%CreateTable(
ds =
,title =
,labelCol =
,columns =
,pValue =
,pFmt = pvaluef.
,footnote =
,cWidth1 = 350
,cWidth2 = 250
,outputWidth =
,fontSize = 11
,fontStyle = Roman
,fontFace= Times
,boldedLabel = 0
,boldedSigP = 1
,sigP = 0.05
,style = rtf
,newPage = 0
,page = portrait
,rtfFile =
,xmlFile =
,pdfFile =
);
Returns
A table produced by proc report
Note
  • It is recommended that all variables be formatted and labeled
  • Unless running on SAS Studio, output might not look right in results window but styling in RTF file will be fine
  • On occasion you might see an error message about the width of a column not being between 1 and 78. This has to do with the listing destination being open and does not affect the output. If you wish to save an error-free copy of your log, use "ods listing close;" prior to calling the macro.
Parameters
dsThe data set to be used to generate table
labelColOptional name of a single variable to be used as the label column; column will be left justified
columnsList of 1+ variable names to include in table; columns will be center justified
titleOptional title for table; should be surrounded by double quotes
pValueThe numeric variable that contains the p-vlaues; column will be center justified
pFmtFormat for p-value column
footNoteOptional footnote to be included under the table; should be surrounded by double quotes
cWidth1Specify width of label column in pixels, default is 350
cWidth2Specify width of columns in pixels
Default is 300
outputWidthSpecify output width of the table in terms of % of RTF page; value should be 1-100 with 100 occupying 100% of the page
Default size is determined by PROC REPORT
fontSizeFont size in pt
Default is 11
fontFaceSpecify font face
Default is Times
fontStyleSpecify font style
Default is Roman
boldedLabelSpecify whether label column uses bold font
Allowed options are:
0 = Use normal font (DEFAULT)
1 = Use bold font
boldedSigPSpecify whether significant p-values use bold font
Allowed options are:
0 = Use normal font
1 = Use bold font (DEFAULT)
sigPSpecify significance level for bold p-values
Default is 0.05
styleODS style to be used
Default is rtf
newPageSelect to start new page; this only takes effect if inluding in a larger ODS output file and not using the rtfFile, xlmFile or pdfFile options within the macro
Allowed options are:
0 = No, continue on same page
1 = Yes, start new page (DEFAULT)
pageSpecify page orientation for ods. If Landscape used, user must start new page in next section/figure/table call to revert to portrait
Allowed options are:
portrait (DEFAULT)
landscape
rtffileOptional RTF name in which to store the output, it should be surrounded in quotes, e.g. "~userid/consult/out.rtf"
xmlfileOptional Excel XML file name in which to store the output, it should be surrounded in quotes, e.g. "~userid/consult/out.xml"
This has some stylying issues and is not recommended
pdffileOptional PDF file name in which to store the output, it should be surrounded in quotes, e.g. "~userid/consult/out.pdf"
Example(s)
proc reg data=sashelp.class;
model weight = height age;
ods output parameterEstimates=pe;
run;
%CreateTable(
ds = pe,
title = "Example",
labelCol = Variable,
columns = estimate stdErr,
pValue = probT,
footnote = "My footnote here");
