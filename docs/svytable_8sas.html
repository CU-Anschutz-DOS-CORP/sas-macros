Detailed Description

This program computes summaries for one or more group(s) and test statistics for differences between two (or more) independent samples using SAS SURVEY procedures. It can output well-formatted tables into RTF files.

Usage
A call to this macro has some or all of these parameters (defaults given):
%svytable(
data = , class = ,
domain = , strata = , cluster = , weight = ,
con1 = , con3 = , cat1 = , cat2 = ,
alpha = 0.05, colpct = 1, secl = se, seopt = 1, oddsratios = 0,
pvalues = 1, totalcol = 1, ncol = 0, misscol = 0,
desclabel = 1, boldlabel = 0, boldsigp = 1, printptype = 1,
lablen = $100., outputwidth = , tbltitle = , sortby = , list = ,
f1 = __countf., f2 = __estf., pfmt = pvaluef., orfmt = __estf.,
cwidth1 = 85, cwidth2 = , cwidth3 = ,
out = _out_, rtfout = _rtf_,
rtffile = , style = journal, page = portrait,
bdytitle = 0, daytime = 0, debug=0);
Returns
SAS output listing (if running in SAS Studio) and data sets of summary statistics, as well as an RTF file, if requested
Parameters
dataThe name of the input dataset to be used
domainDomain variable that identifies the subset of interest
The domain of interest must be coded as 1
By default all records will be used
strataVariable that defines sampling strata
clusterVariable that defines sampling clusters
weightVariable that defines sampling weight
con1List of continouos variables to summarize as mean +/- SE or CL and compare using linear regression
con3List of continouos variables to summarize as median [Q1, Q3] and compare using linear regression with log transformation
cat1List of binary variables to summarize as % (SE or CL) and compare using Rao-Scott chi-square
Variables should be numeric and coded 0/1 or 1/2; only level=1 will be displayed
cat2List of nominal variables to summarize as % (SE or CL) and compare using Rao-Scott chi-square
Can be numeric or character; all levels will be displayed
classThe variable which defines the groups
Optional; must be a numeric variable
alphaSpecify overall significance level
This will be used to bold p-values and to calculate post-hoc significance criterion, if applicable
Default is 0.05, value should be between 0 and 0.99
colpctChoose between column or row percentages for cat1, cat2 or ord1 variables
Allowed options are:
0 = Row percent
1 = Column percent (DEFAULT)
oddsratiosInclude unadjusted odds ratios when class variable has 2 groups.
These are calculated with CLASS var as the outcome and the following options: order=internal, event descending, ref=first
Allowed options are:
0 = No, do not include ORs (DEFAULT)
1 = Yes, include ORs
seclChoose between standard error or 1-alpha% confidence interval
Allowed options are:
SE = standard error (DEFAULT)
CL = confidence limits
cltypeChoose method to calculate confidence limits for proportions
Allowed options are: WALD (DEFAULT), CLOPPERPEARSON, CP, LOGIT, WILSON, SCORE
seoptChoose how the standard error should be displayed
Allowed options are:
1 = with +/- sign (DEFAULT)
2 = in parentheses
pvaluesInclude p-values for group comparisons when 2 or more groups in CLASS var
Allowed options are:
0 = No, only include summary statistics
1 = Yes, include p-values (DEFAULT)
totalcolInclude Overall summary statistics when class variable specified
Allowed options are:
0 = No, do not include overall columns
1 = Yes, include overall columns (DEFAULT)
ncolInclude a column with N for each variable
Allowed options are:
0 = No N column included (DEFAULT)
1 = Overall N
2 = N Per class level
3 = Both Overall and per group Ns
misscolInclude a column with n missing for each variable
If ncol is also specified, misscol will be ignored
Allowed options are:
0 = No n missing column included (DEFAULT)
1 = Overall n missing
2 = Missing per class level
3 = Both Overall and per group missing
desclabelUse variable label with information on values presented [e.g. Male, No. (%) (instead of Male)]
Allowed options are:
0 = No, use variable label only (DEFAULT)
1 = Yes, use more descriptive label
BoldedLabelSpecify whether label column should use bold font
Allowed options are:
0 = No, use normal font (DEFAULT)
1 = Yes, use bold font
printptypeDenote type of test used with a superscript for each p-value @Regardless of choice, a footnote with definition of superscripts will be printed
Allowed options are:
0 = No, do not denote type of test
1 = Yes, denote type of test (DEFAULT)
BoldedSigPSpecify whether significant p-values use bold font
You can use ALPHA option to set your significance level
Allowed options are:
0 = No, use normal font
1 = Yes, use bold font for significant p (DEFAULT)
labLenMaximum Label length captured
Default is $100.
tbltitleA title for table surronded by double quotes
addfnOptional user-defined footnote surrounded by double quotation marks
sortbyChoose how to sort rows in final output
Allowed options are:
list : Sort by the list given (see LIST)
pval : Sort by the p-value
varname : Sort by variable names
<blank> : In input order for CON1, CON2, CON3, CAT1, CAT2, ORD1 (DEFAULT)
listList the variables in the order they should be reported when SORTBY=LIST
If you do not include all variables, the rest will be included in input order
f1Format applied to counts (n) in RTFOUT
Default is __COUNTF.
f2Format applied to mean, SE, medians, P25, P75, %, CL in RTFOUT
Default is __ESTF.
orFmtFormat applied to OR and CL in RTFOUT
Default is __ESTF.
pFmtFormat applied to p-values in RTFOUT
Default is PVALUEF.
cwidth1Column width for N total/N missing/p-value columns
Default is 85
cwidth2Column width for summary stats columns
Default is 200 when CL is requested, 175 otherwise
cwidth3Column width for variable labels
Default is 375 when desclabel=1 or any CON3 variables included, 350 when desclabel=1 without con3, and 300 otherwise
outName for the data set in which the results are stored
Default is OUT
rtfoutThe data set in which results for RTF printing are stored
Default is RTF
rtffilePath and filename for optional RTF output file
It should be surrounded in quotes, e.g. "~userid/consult/out.rtf"
styleDefine a style for ods output
Default is JOURNAL
pageSpecify page orientation for output files
Allowed options are:
PORTRAIT (DEFAULT)
LANDSCAPE
outputwidthSet percentage of the page the table occupies in the output report
Value must be 1-100 and if not specified PROC REPORT will decide best value
bdytitleLocation of SAS system titles/footnotes on RTF file
Allowed options are:
0=Put Titles in Header and Footnotes in Footer (DEFAULT)
1=Put Main Title and Footnotes in the Body of the RTF file
daytimeAdd a date/time stamp as a footnote to the ods files.
Allowed options are:
0 = No date stamp (DEFAULT)
1 = Yes, include date/time stamp
debugToggle between debugging options on/off
Allowed options are:
0 for Off (DEFAULT)
1 for On
Note
  • It is a good idea to have all variables formatted, especially the CLASS variable
  • You can sort the results by any combination of variables found in the &OUT dataset and use the DESCENDING option as well
  • CAT2 may contain SAS numeric or character variables, all other variable lists should contain SAS numeric variables
  • The RTFOUT dataset is still created even if RTFFILE is blank
  • Variables which have all missing values will not be analyzed; a warning of this occurrence is sent to the SAS log
  • Created data sets that are NOT deleted: &OUT &RTFOUT
  • If NTOTALCOL=0, variable labels are flagged if there are missing values and a footnote is added stating # for each var
  • BDYTITLE parameter only works with RTF files
  • If no class variable specified, TotalCol is ignored. If you want to print the N or N miss columns use NCOL=2 or MISSCOL=2 otherwise it will be ignored
  • If running the interactive SAS in Linux, no output will be seen on the listing window but the RTF file will be created
Example
filename xptIn url "https://wwwn.cdc.gov/nchs/nhanes/2015-2016/DEMO_I.xpt";
libname xptIn xport;
data demo_i;
set xptIn.demo_i;
analysisSubset = (RIDAGEYR ge 18); **adults;
run;
libname xptIn clear;
%svytable(
data = demo_i
,class = RIAGENDR
,domain = analysisSubset
,strata = SDMVSTRA
,cluster = SDMVPSU
,weight = WTINT2YR
,con1 = INDFMPIR
,cat2 = RIDRETH1 DMDEDUC2
,tbltitle = "Example of SVYTABLE"
,rtffile = "./exampleSvyTable.rtf"
);
