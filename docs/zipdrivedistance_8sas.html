Detailed Description

Purpose
This macro accesses Google Maps and returns the driving distance between two 5-digit zip codes
Author
Rocio Lopez
Date
10-15-2018
Version
SAS 9.4
Usage
A call to this macro has some or all of these parameters (defaults given):
%zipDriveDistance(
data=
,zipcode1=
,zipcode2=
,outdata =
,straightline=1
,debug=0
);
Returns
A sas data set with columns added for driving distance, driving distance units, driving time, and, if requested, straight line distance
Note
  • This is based on SGF Paper 050-2010 and this SAS communities post
  • Zip codes should be 5 digits but can be numeric or character
  • Finding the desired values of distance within the web page produced by Google Maps is based on the current structure of those web pages; these structure is subject to change at any time and this macro will probably need to be updated to accommodate any changes
  • Takes 1-3 seconds per query (observation)
  • The user should revise observations without reported distances as sometimes they are returned as blank but a manual check of Google will give a distance
Parameters
dataName of SAS dataset containing zip codes
zipCode1Name of column with zip code of origin
zipCode2Name of column with destination zip code
straightLineInclude the straight line distance (ZIPCITYDISTANCE function)?
Allowed options are:
0 for no
1 for yes (DEFAULT)
outDataName of new SAS dataset that will have original data + driving distance
If not provided, default will be <data>_drive_distance
debugDebugging on/off
Allowed options are:
0 for off (DEFAULT)
1 for on
Example(s)
data zip_info;
input z1 @@;
if z1=00966 then z2=00969;
else z2=12203;
format z1 z2 z5.;
datalines;
02138 13502 20037 94117 12144 12203 00966
;
run;
%zipDriveDistance(
data=zip_info
,zipcode1=z1
,zipcode2=z2
);
proc print data=zip_info_drive_distance; run;
