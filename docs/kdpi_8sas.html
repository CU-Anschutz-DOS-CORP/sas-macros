Detailed Description

Purpose
This macro runs within a DATA step and calculates the Kidney Donor Profile Index (KDPI)
Author
Rocio Lopez
Date
07-21-2023
Version
SAS 9.4 or later
Usage
A call to this macro has all or some of these parameters:
%kdpi(
donType =
,age =
,hgt_cm =
,wgt_kg =
,black =
,htn =
,diabetes =
,codCVA =
,creatinine =
,hcv =
,dcd =
,scalingFactor =
,kdpiInformat =
,prefix =
);
Returns
The following columns will be added to the data set being created: kdri_rao, kdri_med, and kdpi
Parameters
donTypeVariable specifying donor type coded as C for cadaveric or L for living
ageVariable containing donor age
hgt_cmVariable containing donor height in centimeters
wgt_kgVariable containing donor weight in kilograms
blackVariable indicating whether the donor is of Black race
This must be coded as 1 = black and 0 = other race
htnVariable indicating whether donor has history of hypertension
This must be coded as 1 = yes and 0 = no
diabetesVariable indicating whether donor has history of diabetes
This must be coded as 1 = yes and 0 = no
codCVAVariable indicating whether the donor died due to CVA or no
This must be coded as 1 = CVA and 0 = other cause of death
creatinineVariable containing donor serum creatinine in mg/dL
hcvVariable indicating whether the donor is HCV positive
This must be coded as 1 = yes and 0 = no
dcdVariable indicating the donor DCD status
This must be coded as 1 = DCD and 0 = DBD

@scalingFactor A single number representing the scaling factor for converting KDRI RAO to scaled KDRI
This is obtained from the KDPI mapping table for the reference population of choice.

@kdpiInformat The name of the informat that contains the KDPI mapping table for the reference population of choice.

Parameters
prefixOptional prefix for variable names
For example, if prefix = d_ then the 3 variables created will be d_kdpi_rao d_kdpi_med d_kdpi
Note
  • This macro will only run from within a data step.
  • KDPI is only calcualted for cadaveric donors donTyp='C'
  • The user must have defined an informat with the KDPI mapping table of choice.
  • KDRIrao (kdri_rao) is interpreted as the relative risk of post-transplant graft failure for this donor compared to a reference donor (age=40 years, non-African American, etc.) as defined in Rao, et al.
  • KDRImedian (kdri_med) is interpreted as the relative risk of post-transplant graft failure (in an average, adult recipient) for this donor compared to the median kidney donor recovered in the reference population year.
  • KDRI is a measure of relative risk of post-transplant graft failure in a cumulative percentage scale.
  • Example interpretation: The estimated risk of kidney graft failure from this donor is higher than <KDPI>% of kidney donors recovered in <reference population year> and <KDPImedian> times that of the median donor recovered in <reference population year>.
  • Guide to calculating KDPI: https://optn.transplant.hrsa.gov/media/j34dm4mv/kdpi_guide.pdf
  • Online KDPI calculator: https://optn.transplant.hrsa.gov/data/allocation-calculators/kdpi-calculator/
    Note that the calculator uses the prior year as the reference population (e.g. if you use it in 2023, it uses 2022 as the reference population)
  • Archive of KDPI Mapping Tables: https://optn.transplant.hrsa.gov/data/allocation-calculators/archived-allocation-calculator-resources
Example(s)
*Ref pop used in example: All kidney donors recovered during 2022;
*https://optn.transplant.hrsa.gov/media/wnmnxxzu/kdpi_mapping_table.pdf;
*Import 2022 KDPI mapping informat;
%include "./optnMappingTables/kdpi_mapping_2022.sas";
*Input data and calculate KDPI;
title "KDPI";
data kdpi;
retain d_type 'C';
input d_age d_hgt_cm d_wgt_kg d_black d_htn d_diabetes d_codCVA d_scr d_hcv d_dcd;
%kdpi(
donTyp = d_type
,age = d_age
,hgt_cm = d_hgt_cm
,wgt_kg = d_wgt_kg
,black = d_black
,htn = d_htn
,diabetes = d_diabetes
,codCVA = d_codCVA
,creatinine = d_scr
,hcv = d_hcv
,dcd = d_dcd
,scalingFactor = 1.33586831546044
,kdpiInformat = inKdpi22F.
,prefix = d_
);
label d_kdpi_rao = "Donor KDPI RAO"
d_kdpi_med = "Donor KDPI Median"
d_kdpi = "Donor KDPI (%)";
datalines;
52 183 81 0 1 0 1 1.7 0 1
;
run;
proc print data=kdpi; *label;
run; title;
