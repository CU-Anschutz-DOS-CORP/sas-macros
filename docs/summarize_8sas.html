Detailed Description

This program summarizes multiple variables for one or more independent samples and generates a report-ready table. In the case of two or more groups, test statistics for differences between groups are included.

Author
Rocio Lopez
Date
02-04-2019
Version

Usage
A call to this macro has some or all of these parameters (defaults given):
%summarize(
data=, subset=, class=,
con1=, con2=, con3=, cat1=, cat2=, ord1=,
sdopt=1, unequalvar=,
colpct=1, cutexact=20,
pvalues=1, alpha = 0.05,
adhoc=0, adhoc_method=bon,
oddsratios=0, unitscon1=, unitscon2=, unitscon3=,
out=_out_, rtfout=_rtf_,
sortby=, list=,
f1= __Countf., f2= __Estf., f3= __Estf., f4= __Estf., pfmt= Pvaluef.,
tbltitle=, totalcol=1, ncol=0, misscol=0, desclabel=0, BoldedLabel=0,
printptype=1, boldedsigp=1, daytime=0, printfn=1, addfn=,
rtffile=, pdffile=, xlsxfile=, xmlfile=, bdytitle=0, page=portrait,
cwidth1=, cwidth2=, cwidth3=, fontsize=11, fontface=Times, outputWidth=,
debug=0);
Returns
SAS output listing (if running in SAS Studio) and data sets of summary statistics, as well as RTF, PDF and/or XLSX files, if requested.
Parameters
dataThe name of the input dataset to be used
con1List of continouos variables to summarize as mean +/- SD and compare using t-tes or ANOVA
con2List of continouos variables to summarize as median (min, max) and compare using Wilcoxon Rand Sum or Krukal-Wallis tests
con3List of continouos variables to summarize as median [Q1, Q3] and compare using Wilcoxon Rand Sum or Krukal-Wallis tests
cat1List of binary variables to summarize as n (%) and compare using Pearson's chi-square or Fisher's Exact test
Only level=1 will be displayed
cat2List of nominal variables to summarize as n (%) and compare using Pearson's chi-square or Fisher's Exact test
All levels will be displayed
ord1List of ordinal variables to summarize as n (%) and compare using Wilcoxon Rand Sum or Krukal-Wallis tests
classThe variable which defines the groups
Optional; must be a numeric variable
subsetOptional expression to be used in a WHERE statement to in order to subset the data set.
sdoptAllows to choose how the standard deviation should be displayed
Allowed options are:
1 = with +/- sign (DEFAULT)
2 = within parentheses
unequalvarChoose what types of tests are performed for variables in CON1 list
Allowed options are:
0 = Equal variances (t-test or ANOVA) (DEFAULT if CLASS var has 3+ groups)
1 = Unequal variances (Satterthwaite t-test or Welch's ANOVA)
2 = Let the macro determine which to use based on tests of equality of variances (DEFAULT if CLASS var has 2 groups)
colpctChoose between column or row percentages for cat1, cat2 or ord1 variables
Allowed options are:
0 = Row percent
1 = Column percent (DEFAULT)
cutexactFisher's Exact test is performed when the % of cells with expected counts<5 in categorical vars is greater to or equal to CUTEXACT.
Default is 20, value should be between 0 and 100
pvaluesInclude p-values for group comparisons when 2 or more groups in CLASS var
Allowed options are:
0 = No, only include summary statistics
1 = Yes, include p-values (DEFAULT)
alphaSpecify overall significance level
This will be used to bold p-values and to calculate post-hoc significance criterion, if applicable
Default is 0.05, value should be between 0 and 0.99
adhocPerform all possible pairwise post-hoc comparisons when CLASS has 3+ groups
This is only done for variables with a significant overall p-value
Allowed ptions are:
1 = Yes, perform pairwise comparisons
0 = No, do not perform pairwise comparisons (DEFAULT)
adhoc_methodMethod used to adjust pair-wise comparisons
Several options from PROC MULTTEST is allowed, see PROC MULTTEST documentation for details
Resampling-based adjustments are not permitted
Default is: BON
oddsratiosInclude unadjusted odds ratios when class variable has 2 groups.
These are calculated with CLASS var as the outcome and the following options: order=internal, event descending, ref=first
Allowed options are:
0 = No, do not include ORs (DEFAULT)
1 = Yes, include ORs
unitscon1If odds ratios requested, specify units for CON1 variables
If you specify 1 single number, it will be used for all variables
If you specify >1 numbers, you must include 1 number per CON1 variable and in the the same order as the CON1 list
Default is 1 for all variables
unitscon2If odds ratios requested, specify units for CON2 variables
If you specify 1 single number, it will be used for all variables
If you specify >1 numbers, you must include 1 number per CON2 variable and in the the same order as the CON2 list
Default is 1 for all variables
unitscon3If odds ratios requested, specify units for CON3 variables
If you specify 1 single number, it will be used for all variables
If you specify >1 numbers, you must include 1 number per CON3 variable and in the the same order as the CON3 list
Default is 1 for all variables
outName for the data set in which the results are stored
Default is OUT
rtfoutThe data set in which results for RTF printing are stored
Default is RTF
sortbyChoose how to sort rows in final output
Allowed options are:
_list : Sort by the list given (see LIST)
pval : Sort by the p-value
varname : Sort by variable names
varlabel: Sort by variable label
<blank> : In input order for CON1, CON2, CON3, CAT1, CAT2, ORD1 (DEFAULT)
listList the variables in the order they should be reported when SORTBY=_LIST
If you do not include all variables, the rest will be included in input order
You can include section headers in the LIST parameter
Each section header must start with TblHdr_ and be ≤ 25 characters in length; use underscore (_) instead of space
f1Format applied to counts (n) in RTFOUT
Default is __COUNTF.
f2Format applied to mean, SD, medians, P25, P75, min, max in RTFOUT
Default is __ESTF.
f3Format applied to percents (underlying value is 0-100) in RTFOUT
Default is __ESTF.
f4Format applied to OR and 95% CI in RTFOUT
Default is __ESTF.
pfmtFormat applied to p-values in RTFOUT
Default is PVALUEF.
tbltitleA title for table surronded by double quotes
totalcolInclude Overall summary statistics when class variable specified
Allowed options are:
0 = No, do not include overall columns
1 = Yes, include overall columns (DEFAULT)
ncolInclude a column with N for each variable
Allowed options are:
0 = No N column included (DEFAULT)
1 = Overall N
2 = N Per class level
3 = Both Overall and per group Ns
misscolInclude a column with n missing for each variable
If ncol is also specified, misscol will be ignored
Allowed options are:
0 = No n missing column included (DEFAULT)
1 = Overall n missing
2 = Missing per class level
3 = Both Overall and per group missing
desclabelUse variable label with information on values presented [e.g. Male, No. (%) (instead of Male)]
Allowed options are:
0 = No, use variable label only (DEFAULT)
1 = Yes, use more descriptive label
BoldedLabelSpecify whether label column should use bold font
Allowed options are:
0 = No, use normal font (DEFAULT)
1 = Yes, use bold font
printptypeDenote type of test used with a superscript for each p-value @Regardless of choice, a footnote with definition of superscripts will be printed
Allowed options are:
0 = No, do not denote type of test
1 = Yes, denote type of test (DEFAULT)
BoldedSigPSpecify whether significant p-values use bold font
You can use ALPHA option to set your significance level
Allowed options are:
0 = No, use normal font
1 = Yes, use bold font for significant p (DEFAULT)
printfnInclude default footnotes in table
Allowed options are:
0 = No, do not include
1 = Yes, include (DEFAULT)
addfnOptional user-defined footnote surrounded by double quotation marks

Allowed options are:
0 = No date stamp (DEFAULT)
1 = Yes, include date/time stamp

It should be surrounded in quotes, e.g. "~userid/consult/out.rtf"

It should be surrounded in quotes, e.g. "~userid/consult/out.pdf"
xlsxfilePath and filename for optional XLSX output file
It should be surrounded in quotes, e.g. "~userid/consult/out.xlsx"

It should be surrounded in quotes, e.g. "~userid/consult/out.xml"

Allowed options are:
0=Put Titles in Header and Footnotes in Footer (DEFAULT)
1=Put Main Title and Footnotes in the Body of the RTF file

Allowed options are:
PORTRAIT (DEFAULT)
LANDSCAPE

Default is 95 if printptype=0 and 90 otherwise
cwidth2Column width for summary stats columns
Default is 275 when CON2 or CON3 are used, 175 otherwise

Default is 375 when desclabel=1 and 275 otherwise

Note
  • At least one list of variables (CON1, CON2, CON3, CAT1, CAT2, ORD1) must be provided.
  • It is a good idea to have all variables formatted, especially the CLASS variable
  • CAT2 may contain SAS numeric or character variables, all other variable lists should contain SAS numeric variables.
  • A variable can occur in more than one display parameter list. If doing so, it is recommended you use the DESCLABEL=1 option.
  • Variables which have all missing values will not be analyzed. Notification of this occurrence is sent to the SAS log.
  • If NTOTALCOL=0, variable labels are flagged if there are missing values and a footnote is added stating # missing for each var.
  • BDYTITLE parameter only works with RTF and XML files
  • Default is to perform Exact tests for categorical variables if more than 50% of class have counts less than 5 but this can be manipulated with the CUTEXACT option. To suppress exact tests then set CUTEXACT to a value greater than 100. A value of 0 would produce Exact tests for all variables.
  • If no class variable specified, TotalCol is ignored. If you want to print the N or N miss columns, use NCOL=2 or MISSCOL=2 otherwise it will be ignored. *3/20/2020: Now either 1 or 2 will work.
  • You can list variables with the X1-X99. The X: or FirstCol – LastCol syntax are not supported.
  • You can include section headers in the LIST parameter. These will be included in the label column. Each section header must start with TblHdr_ and be ≤ 25 characters in length. Use an underscore (_) instead of a space. To insert blank lines use TblHdr__ (2 underscores).
  • If running on interactive Linux SAS, no output is listed on results window. OUT and RTFOUT are generated and if you specify an output file that will be created.
  • Unless running on SAS Studio, output might not look right in results window but styling in RTF file will be fine
  • When comparing 3+ groups, it is recommended that the user make the decision of which type of ANOVA (equal vs. unequal variances) to use. Homogeneity of variance tests have too little power to be relied upon to always detect when Welch’s ANOVA is appropriate. Unless the group variances are extremely different or the number of groups is large, the usual ANOVA test is relatively robust when the groups are all about the same size. Graphical diagnostics can be a useful informal tool for monitoring whether your data meet the assumptions of a GLM analysis.
  • OUTPUTWIDTH option does not work with PDF and should not be changed if requesting a PDF file.
  • Data sets (potentially) created: ngroups vtype n order con1 tcon1 con2 tcon2 con3 tcon3 cat1 cat2 ord1 keepcat: p_info ttest ttest2 eqvar anova: welch: parp: npar: fish1-_fish3 _csq fisher chisq: outds outcon1 outcon2 outcon3 outcat oddsratios summ pval adhoc_pval adhoc_pval2 add tout tempd _f
  • Created data sets that are NOT deleted: &OUT &RTFOUT

Example
title1 'Sample execution of SUMMTABLE';
footnote "Randomly generated data";
proc format;
value groupf 1="Group A" 2="Group B";
value x5f 0="Failure" 1="Success";
run;
data a;
do Group=1 to 2;
do i=1 to 100;
X1 = rannor(0);
X2 = 100*ranexp(0);
X3 = exp(rannor(0));
X4 = ranbin(0, 1, 0.5);
X5 = ranbin(0, 1, 0.7);
X6 = ranbin(0, 4, 0.8);
output;
end;
end;
label X1="Normal variate"
X2="Exponential variate"
X3="Log-normal variate"
X4="Bernoulli variate"
X5="Formatted Bernoulli"
X6="Ordinal variate";
format group groupf. X5 x5f.;
run;
%summarize(data=a
,class=group
,rtffile="summary.rtf"
,page=landscape
,con1=x1
,con2=x2
,con3=x3
,cat1=x4
,cat2=x5
,ord1=x6
,sortby=_list
,list=x2 x1 x6
,tbltitle="Data summaries"
);
